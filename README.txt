
--------------------------------Содержание папки-------------------------------------

conf_files | папка с yml-файлами для настройки сценария расчета
contours | папка с txt-файлами из координат контуров
files | папка с исходным файлом
output | папка для сохранения графиков и отчета в .xlsx формате
Опорная сеть.exe | приложение для построения опорной сети

-----------------------------------Инструкция----------------------------------------

1) Выгрузить исходный файл с данными по всем скважинам месторождения из NGT Smart 5.8.2 в формате Excel.xlsx. Проверить, что столбцы координат устья и забоя расположены в правильном порядке:
координаты устья, затем координаты забоя
Для расчета используются следующие столбцы:
№ скважины: wellName,
Дата: nameDate,
Характер работы: workMarker,
Состояние: wellStatus,
Объекты работы: workHorizon,
Координата X: coordinateXT1,
Координата Y: coordinateYT1,
Координата забоя Х (по траектории): coordinateXT3,
Координата забоя Y (по траектории): coordinateYT3,
Дебит нефти (ТР), т/сут: oilRate,
Приемистость (ТР), м3/сут: injectivity

2) Открыть папку well_net. До запуска main.exe необходимо настроить расчет в файле parameters.yml.

2.1) ПУТИ К ФАЙЛАМ И ПАПКАМ:

- data_file содержит путь к исходному файлу с данными
- gdis_file сожержит путь к файлу с данными по проведенным ГДИС

2.2) ИЗМЕНЯЕМЫЕ ПАРАМЕТРЫ РАСЧЕТА:

- mult_coef является списком коэффициентов кратного увеличения среднего радиуса, рассчитанного на объект. Для каждого
коэффициента производится отдельный расчет
- limit_radius_coef ограничение на коэффициент или максимальный коэффициент домножения радиуса, больше которого
появляются скважины, которые не представляется возможным исследовать, так как они перекрываются другими, расположенными
ближе к исследуемой скважине. Таким образом, все коэффициенты в списке больше ограничения, будут подаваться на
отдельный сценарий расчета, где будет учитываться дообследование скважин, попавших в "слепые" зоны
- percent параметр, задающий минимальный процент длины тректории скважины для включения ее в зону охвата исследуемой
скважины. Вертикальные и наклонно-направленные скважины сразу же попадают в зону охвата, так как являются точками при 
расчете
- gdis_option параметр, задающийся в годах. Определяет, какой давности ГДИС из соответствующего файла gdis_file будут 
взяты исследования и каких скважин. Далее в расчете эти скажины будут удалены из исходных данных data_file.
Рекомендуемые значения параметра: "нет" - ГДИС в расчете не учитываются; 1, 2, 3... - учитываются ГДИС,
проведенные такое количество лет назад
- separation_by_years задает количество лет, на которые нужно распределить скважины, попавшие в "слепые" зоны, то есть 
при домножении исходного радиуса на коэффициент больше, чем limit_radius_coef. Рекомендуемые значения параметра: 
"нет", 1, 2
- calc_option определяет сценарий вхождения скважины в зону охвата(True - скважина включается в зону, если
определенный процент ее траектории входит в эту зону, False - скважина включается в зону охвата при попадании забоя
в эту зону
- verticalWellAngle угол для обозначения зоны вертикальных скважин на карте в полярной системе координат
- MaxOverlapPercent максимальный процент перекрытия от общей длины скважины, чтобы она вошла в первый ряд окружения
- angle_horizontalT1 угол расширения сектора для точки Т1 горизонтальной скважины
- angle_horizontalT3 угол расширения сектора для точки Т3 горизонтальной скважины

2.3) ОПТИМАЛЬНО ЗАДАННЫЕ ПАРАМЕТРЫ(НЕ РЕКОМЕНДУЕТСЯ ИЗМЕНЯТЬ):

- min_length_horWell 150 минимальная длина горизонтальной скважины
- max_distance 1000 максимальное расстояние между скважинами

3) Запустить main.exe и дождаться завершения расчета.

4) В папке output/pictures будут находиться полученные в процессе расчета картинки, в папке output
сохранится .xlsx файл с отчетами по каждому контуру и кратким общим отчетом, а также .txt файл с логированием.
Последний необходимо скинуть разработчикам в случае возникновения ошибок при расчете.
